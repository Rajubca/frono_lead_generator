Frono AI Lead Generator Documentation

This document provides a comprehensive technical overview of the frono_lead_generator repository, detailing function-level logic and variable management.

1. app.py — Orchestration Layer

The core API handling session state, intent routing, and order execution.

Key Global Variables

  • user_sessions: Dictionary mapping session_id to state objects containing history, current stage, and email.
  • stock_reservations: Cache holding SKU, name, and price for items validated during the checkout phase.
  • user_queues: Thread-safe queues for real-time streaming responses.

Core Functions

process_message(req: PromptRequest)
  • Intent & Contact: Extracts user intent and email/phone details.
  • Strict Digit Matching: Resolves numeric inputs (e.g., "2") into full product names via session["menu"].
  • SKU Locking: Saves session["last_sku"] to ensure the agent maintains focus on a specific product during checkout.
chat_stream(req: PromptRequest, background_tasks)
  • Flow: Generates AI tokens via LLM and manages SSE delivery.
  • Automation: If the stage is converted, it triggers OpenSearch stock updates and background email tasks.

2. retriever.py — Data Retrieval Layer

Interfaces with OpenSearch to ground the LLM in verified facts.

Core Functions

get_product_by_name(identifier: str)
  • Precision: Uses a term query on sku.keyword for 100% accurate ID matching.
  • Fallback: Uses a fuzzy match query for product names.
retrieve_context(query: str, intent: str)
  • Product Context: Returns a list of available items where qty > 0.
  • Knowledge Context: Searches frono_site_facts for brand and policy data.

3. The Sales Funnel Logic

Users move through distinct stages defined in user_sessions[id]["stage"].

Stage Description Trigger Action
browsing Initial state General category discovery.
interest User asks about a specific item Locks the SKU in the user session.
checkout Buying intent detected Verifies inventory and requests email.
converted Email captured + Stock confirmed Executes stock reduction and email automation.

4. Developer Notes

  • Currency Formatting: Ensure prices are formatted in retriever.py to prevent the LLM from hallucinating decimal placements (e.g., converting 4999 to 49.99).
  • Stock Updates: The client.update method in app.py requires the OpenSearch Document ID to match the sku.
